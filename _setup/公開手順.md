# 本番公開手順（happoh.com）

現在のサイトは **`noindex` + `robots.txt: Disallow: /`** で検索エンジンから遮断されています。
canonical・OGP・sitemap・セキュリティヘッダーは実装済みなので、**下の順番どおり**に進めれば公開できます。

> 順番が大事です。ドメイン接続より先に `noindex` を外すと、`happouhp.pages.dev` の方が
> インデックスされてしまい、あとから直すのが面倒になります。

---

（コミット `fc8b759`。git 履歴に残るため、ファイルから消しただけでは無効化されません）

1. microCMS 管理画面 → **API キー管理**
2. 旧キー `z84ugu…` を **削除（失効）**
3. 新しいキーを発行。権限は **GET のみ**、「下書き参照」は **OFF**
4. 新キーは**どこにもコミットしない**（次の手順で Cloudflare の環境変数に入れる）

## 1. Cloudflare Pages に環境変数を追加

Cloudflare ダッシュボード → Pages → `happouhp` → **Settings → Environment variables**（Production）

| 変数名 | 値 | 必須 |
|---|---|---|
| `GAS_URL` | Apps Script ウェブアプリの `/exec` URL | 設定済み |
| `MICROCMS_API_KEY` | 手順0で発行した**新しい**キー | ✅ 追加する |
| `MICROCMS_SERVICE_DOMAIN` | `happoh`（既定値と同じなら省略可） | 任意 |
| `MICROCMS_ENDPOINT` | `news`（既定値と同じなら省略可） | 任意 |

追加したら **再デプロイ**（Deployments → Retry deployment）してください。

> お知らせは `/api/news` プロキシ経由になりました。キーはブラウザに配信されません。
> `MICROCMS_API_KEY` が未設定だとお知らせは表示されず、トップは静的な仮データのままになります。

---

## 2. happoh.com を Pages に接続

Cloudflare → Pages → `happouhp` → **Custom domains → Set up a domain**

1. `happoh.com` を追加（apex）
2. `www.happoh.com` も追加
3. DNS レコードは Cloudflare が自動作成します（現状 apex に A/CNAME が無い状態）

反映後、`https://happoh.com` が 200 で表示されることを確認。

---

## 3. www → apex の 301 リダイレクト

正規URLは **`https://happoh.com`（www なし）** に統一しています。
Cloudflare ダッシュボード → 対象ゾーン `happoh.com` → **Rules → Redirect Rules** で作成:

- 条件: `Hostname equals www.happoh.com`
- 動作: `Dynamic redirect` → `concat("https://happoh.com", http.request.uri.path)` / 301

（`_redirects` ファイルはパスしか判定できないため、ホスト単位の転送はダッシュボード側で行います）

## 3-b.（推奨）pages.dev → happoh.com の 301

`happouhp.pages.dev` が検索結果に出ないよう、同様に Redirect Rule を作成:

- 条件: `Hostname equals happouhp.pages.dev`
- 動作: 301 → `https://happoh.com` + パス

> canonical タグでも重複は防げますが、301 の方が確実です。

## 4. 検索エンジンへ公開する（ここで初めて noindex を外す）

`happoh.com` が正しく表示されることを確認してから実施してください。

### 4-1. 各ページの `noindex` を削除（11ファイル）

対象: `index.html` / `八宝の想い.html` / `事業内容.html` / `会社概要.html` /
`採用情報.html` / `採用-grancha.html` / `採用-umasshu.html` / `採用-muginokura.html` /
`採用-kamameshi.html` / `お知らせ.html` / `プライバシーポリシー.html`

各ファイルの以下2行を削除します:

```html
<!-- 公開手順: _setup/公開手順.md 参照。happoh.com 接続後にこの meta を削除する -->
<meta name="robots" content="noindex,nofollow">
```

> `404.html` と `八宝コーポレートサイト.html`（旧URLのリダイレクト用）は
> **noindex のまま残す**のが正解です。

### 4-2. `robots.txt` を公開用に差し替え

```
User-agent: *
Allow: /
Disallow: /api/

Sitemap: https://happoh.com/sitemap.xml
```

### 4-3. コミット & push

`main` に push すると Cloudflare Pages が自動デプロイします。

## 6. 公開後の確認

```bash
# noindex が消えているか（0件になるはず）
curl -s https://happoh.com/ | grep -c noindex

# robots.txt が開放されているか
curl -s https://happoh.com/robots.txt

# sitemap が引けるか
curl -sI https://happoh.com/sitemap.xml | head -1

# セキュリティヘッダーが付いているか
curl -sI https://happoh.com/ | grep -iE 'content-security-policy|strict-transport|x-content-type|referrer-policy'

# お知らせAPIが動くか（キーは露出しない）
curl -s "https://happoh.com/api/news?limit=1" | head -c 200
```

ブラウザの DevTools → Console に CSP 違反が出ていないかも確認してください。

## 残タスク（任意）

- **求人構造化データ（JobPosting）**: Google しごと検索に載せられます。
  実装には各求人の **掲載開始日（datePosted）** と **募集終了予定日（validThrough）** が必要なので、
  値が決まったら追加できます。採用ページのSEOとして効果が大きいです。
- **OGP 画像**: 現在は全ページ共通で `photos/philosophy/team-collage-large.jpg`（1200×840）。
  SNS 最適の 1200×630 を用意すると見栄えが良くなります。ページ別の画像も設定可能です。
- **HSTS の強化**: 現在 `max-age=31536000; includeSubDomains`。
  `preload` を付けてプリロードリストに申請することもできますが、**取り消しが難しい**ため慎重に。
